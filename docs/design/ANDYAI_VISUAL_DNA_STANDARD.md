# AndyAI Visual DNA Standard

## Short definition

Every AndyAI project must have a visual DNA: design rules, visual canon, UI patterns, motion rules, screenshot story rules, and slide/presentation rules.

## Required design documents for future repos

- `/docs/design/DESIGN.md`
- `/docs/design/VISUAL_CANON.md`
- `/docs/design/UI_PATTERNS.md`
- `/docs/design/MOTION_SKILLS.md`
- `/docs/design/SCREENSHOT_STORY.md`
- `/docs/design/SLIDE_CANON.md`

## Canonical TAP-TAP sequence

```text
BOOTSTRAP
→ DESIGN.md
→ VISUAL_CANON.md
→ UI_PATTERNS.md
→ 3–5 visual blocks
→ TAP-TAP APPLY
→ build / commit / tag / push
```

## Meaning

The visual layer is not decoration. It is product comprehension.

If a user cannot see what the product does, the engine remains hidden.  
If the engine remains hidden, the market sees only fog.
