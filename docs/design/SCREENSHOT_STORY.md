# SCREENSHOT_STORY.md — AndyAI Finance Engine

## Purpose

Screenshots should become a story, not a dump of images.

## Story pattern

1. Start with messy input.
2. Show workflow proposal.
3. Show model preview.
4. Show assumptions ledger.
5. Show human approval.
6. Show evidence-backed output.

## Practical rule

Every screenshot must answer: what changed, who reviews it, and what proof remains?
