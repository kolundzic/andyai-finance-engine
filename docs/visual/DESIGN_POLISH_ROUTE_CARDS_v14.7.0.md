# v14.7.0 — Design Polish Route Cards

## Route card doctrine

Each public route should answer three questions:

1. What is this floor?
2. What proof does it show?
3. What is still blocked?

## Card vocabulary

Use:

- verified
- blocked
- read-only
- approval required
- evidence-ready
- no provider connected
- no database writes

Avoid vague language such as:

- smart
- magical
- autonomous
- unlimited
- fully automatic
