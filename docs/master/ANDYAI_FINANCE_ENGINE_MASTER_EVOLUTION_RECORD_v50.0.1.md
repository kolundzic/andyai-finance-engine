---

## 1. Plain-language summary

AndyAI Finance Engine is a governed financial-modeling product surface.

It is not just a chatbot for finance. It is a controlled business system that helps prepare financial-model workflows, explain evidence, expose assumptions, protect human approval, and guide a potential client toward a safe pilot.

---

## 2. Core principle

```text
Proof first.
Human approval above automation.
Execution only after gates.
```

The system may show, explain, preview, and structure.

The system may not secretly execute, write, bill, record, or bypass a human authority gate.

---

## 3. What was built by v50.0.0?

By `v50.0.0`, the project created a public business-system surface:

- public proof and status pages;
- visual fintech-style product pages;
- talk and guided explanation layers;
- client-conversion pages;
- pilot intake surfaces;
- client portal previews;
- evidence and proposal previews;
- pricing, trust, security, roadmap, and demo-theater routes;
- clear safety boundaries for every public route.

---

## 4. Why many versions exist

Each version is one controlled step.

```text
small change → verifier → commit → tag → push → documented milestone
```

This makes the product explainable later to clients, partners, investors, and the future development team.

---

## 5. What is still blocked?

Still blocked:

- live form submit;
- database writes;
- provider API calls;
- service-role usage;
- billing;
- checkout;
- payment;
- microphone access;
- recording;
- hidden network calls.

This is not weakness. This is governance.
