# Canon Lock — v180 Engine Midpoint

## Lock statement

As of v180.2.0, AndyAI Finance Engine has crossed from product proof into engine midpoint maturity.

## Canonical interpretation

The engine is now defined as a governed financial workflow system, not merely a web presentation.

It must always preserve these principles:

1. No blind financial autopilot.
2. No final output without review logic.
3. No claim without evidence linkage.
4. No client handoff without traceable assumptions.
5. No serious finance automation without audit trail.
6. No production promise without build audit.
7. Human approval remains above the machine.

## Operational canon

Fresh clone sandbox is the preferred method for major master batches.

The standard is:

Fresh clone  
→ base tag guard  
→ local commit/tag series  
→ final build gate  
→ push main and tags  
→ final semafor.
